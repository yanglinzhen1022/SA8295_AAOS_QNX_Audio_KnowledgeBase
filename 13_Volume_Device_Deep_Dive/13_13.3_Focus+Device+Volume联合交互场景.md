## 13.3 Focus+Device+Volume联合交互场景

> [← 上一个](13_13.2_Device状态机.md) | [返回13章](README.md) | [返回导航](../README.md) | [下一个 →](13_13.4_外部音频设备-USBHDMI有线设备管理.md)

---

本节深度分析Audio Focus、Device Routing、Volume三个子系统在实际场景中的联合交互机制。Focus决定谁可以播放、Device决定在哪播放、Volume决定播放多大声——三者在AudioService/AudioPolicyManager中紧密联动。

### 13.3.1 三子系统联动架构

```mermaid
flowchart TB
    subgraph Focus子系统
        MFC[MediaFocusControl] --> FOCUS_GAIN[AUDIOFOCUS_GAIN]
        MFC --> FOCUS_LOSS[AUDIOFOCUS_LOSS]
        MFC --> FOCUS_DUCK[AUDIOFOCUS_GAIN_TRANSIENT_MAY_DUCK]
    end
    
    subgraph Device子系统
        BROKER[AudioDeviceBroker] --> DEV_CONN[设备连接/断开]
        BROKER --> DEV_ROUTE[路由切换]
        APM[AudioPolicyManager] --> BEST_DEV[最佳设备选择]
    end
    
    subgraph Volume子系统
        VSS[VolumeStreamState] --> VOL_SET[音量设置]
        SDH[SoundDoseHelper] --> SAFE_VOL[安全音量限制]
        SDH --> CSD_VOL[CSD音量衰减]
    end
    
    FOCUS_GAIN --> MFC_CB[AudioFocusListener回调]
    FOCUS_LOSS --> DUCK[duckFactor计算]
    FOCUS_DUCK --> DUCK
    
    DEV_CONN --> VOL_SYNC[per-device音量同步]
    DEV_ROUTE --> VOL_CURVE[音量曲线切换]
    
    MFC_CB --> AS[AudioService]
    AS --> VSS
    AS --> BROKER
    
    DUCK --> AF_VOL[AudioFlinger.prepareTracks_l]
    SAFE_VOL --> VOL_LIMIT[音量上限限制]
    CSD_VOL --> ATTENUATION[CSD衰减dB值]
    ATTENUATION --> AF_VOL
```

**核心联动原则**:
- **Focus优先**: Focus Gain → 恢复音量; Focus Loss → 降低/静音
- **Device驱动**: 设备切换 → 音量曲线和per-device索引更新
- **Volume受制**: 安全音量/CSD → 可强制降低音量，不受Focus和Device影响

### 13.3.2 Focus→Volume联动：Duck机制

当多个App同时请求Audio Focus时，Focus系统通过Duck机制降低背景App音量，而非完全静音：

```mermaid
sequenceDiagram
    participant NAV as 导航App
    participant MFC as MediaFocusControl
    participant AS as AudioService
    participant AF as AudioFlinger
    
    NAV->>MFC: requestFocus(TRANSIENT_MAY_DUCK)
    MFC->>MFC: 现有MUSIC Focus持有者收到FOCUS_LOSS_TRANSIENT_CAN_DUCK
    MFC->>AS: dispatchFocusChange(LOSS_TRANSIENT_CAN_DUCK)
    AS->>AS: 计算duckFactor
    AS->>AF: setVolume(stream, duckVolume)
    
    Note over AF: prepareTracks_l中:<br>volume = masterVol × streamVol × duckFactor
    AF->>AF: 音乐音量降低到duckFactor倍
    
    Note over NAV: 导航播报结束
    NAV->>MFC: abandonFocus
    MFC->>MFC: MUSIC Focus恢复GAIN
    MFC->>AS: dispatchFocusChange(GAIN)
    AS->>AF: setVolume(stream, normalVolume)
    AF->>AF: 音乐音量恢复正常
```

**duckFactor计算**:

AudioFlinger在[`prepareTracks_l()`](frameworks/base/services/core/java/com/android/server/audio/AudioFlinger.java)中对每条Track计算最终音量：

```
finalVolume = masterVolume × streamVolume × duckFactor × muteFactor
```

| 因子 | 来源 | 范围 |
|------|------|------|
| masterVolume | AudioService.setMasterVolume | 0.0-1.0 |
| streamVolume | VolumeStreamState.applyDeviceVolume | 0.0-1.0 |
| duckFactor | AudioFocus系统 | 0.0-1.0(通常0.1-0.3) |
| muteFactor | VolumeStreamState.mIsMuted | 0或1 |

**Duck阈值**:
- 默认duck factor: 通常为0.2(即音量降低到20%)
- Car Audio场景: CarAudioFocus使用per-zone独立duck
- 导航播报: TRANSIENT_MAY_DUCK → 音乐duck
- 通知提醒: TRANSIENT_MAY_DUCK → 音乐duck

### 13.3.3 Focus→Device联动：通信设备优先

当通信App(电话/VoIP)获取Focus时，Device路由和音量同时切换：

```mermaid
sequenceDiagram
    participant CALL as 电话App
    participant MFC as MediaFocusControl
    participant AS as AudioService
    participant BROKER as AudioDeviceBroker
    participant APM as AudioPolicyManager
    participant VSS as VolumeStreamState
    
    CALL->>MFC: requestFocus(VOICE_CALL, GAIN)
    MFC->>AS: notifyFocusGranted
    
    Note over AS: Focus触发设备路由变化
    AS->>APM: setStreamVolumeIndex(VOICE_CALL, volume, SCO_DEVICE)
    APM->>APM: 路由VOICE_CALL到SCO
    APM->>APM: 路由MUSIC到A2DP或降低优先级
    
    Note over AS: Focus触发音量变化
    AS->>VSS: adjustIndex(VOICE_CALL volume, SCO_DEVICE)
    VSS->>VSS: applyDeviceVolume_syncVSS(SCO_DEVICE)
    
    Note over AS: Focus触发别名映射切换
    AS->>AS: updateStreamVolumeAlias()
    AS->>AS: ALARM别名→ALARM(不再跟随MUSIC)
    AS->>AS: NOTIFICATION别名→NOTIFICATION(独立)
    
    Note over CALL: 电话结束
    CALL->>MFC: abandonFocus
    AS->>AS: restoreAliasMapping(正常模式)
    AS->>AS: restoreDeviceRouting(MUSIC→Speaker/A2DP)
```

**通信模式下的别名切换**:

| Stream | 正常模式别名 | 通话模式别名 | 原因 |
|--------|-------------|-------------|------|
| STREAM_ALARM | STREAM_MUSIC | STREAM_ALARM | 防止闹钟音量被通话duck |
| STREAM_NOTIFICATION | STREAM_MUSIC | STREAM_NOTIFICATION | 防止通知被通话静音 |
| STREAM_DTMF | STREAM_MUSIC | STREAM_MUSIC | DTMF跟随通话音量 |
| STREAM_MUSIC | STREAM_MUSIC | STREAM_MUSIC | 背景音乐被duck |

**设备路由优先级冲突解决**:
- 通话期间: SCO设备优先用于VOICE_CALL策略，MUSIC策略路由到A2DP(如果可用)
- 无A2DP: MUSIC路由到Speaker，但可能被通话duck或完全静音
- AAOS场景: CarAudioService使用Zone隔离，通话Zone和娱乐Zone互不干扰

### 13.3.4 Device→Volume联动：per-device音量

设备切换时，VolumeStreamState的per-device音量索引(mIndexMap)独立维护每个设备的音量值：

```mermaid
sequenceDiagram
    participant BT as 蓝牙耳机
    participant BROKER as AudioDeviceBroker
    participant APM as AudioPolicyManager
    participant AS as AudioService
    participant VSS as VolumeStreamState
    
    Note over BT: 蓝牙耳机连接场景
    BT->>BROKER: A2DP连接
    BROKER->>APM: setDeviceConnectionState(A2DP, AVAILABLE)
    APM->>APM: 路由MUSIC→A2DP
    APM-->>AS: onAudioDeviceListUpdate
    
    AS->>VSS: getIndex(A2DP_DEVICE)
    Note over VSS: mIndexMap中读取A2DP设备的历史音量<br>首次连接→默认安全音量值
    
    AS->>VSS: setIndex(a2dpVolume, A2DP_DEVICE)
    VSS->>VSS: applyDeviceVolume_syncVSS(A2DP_DEVICE)
    Note over VSS: 绝对音量设备→getAbsoluteVolumeIndex换算
    
    AS->>AS: onSetAvrcpAbsoluteVolumeIndex(a2dpVolume)
    Note over AS: A2DP绝对音量双向同步
    
    Note over BT: 蓝牙耳机断开→Speaker场景
    BT->>BROKER: A2DP断开
    APM->>APM: 路由MUSIC→SPEAKER
    AS->>VSS: getIndex(SPEAKER_DEVICE)
    Note over VSS: mIndexMap读取Speaker的历史音量<br>Speaker音量独立于A2DP音量
    
    VSS->>VSS: applyDeviceVolume_syncVSS(SPEAKER)
    Note over VSS: 普通设备→(getIndex+5)/10
```

**per-device音量独立性示例**:

| 设备 | 上次音量索引 | 当前音量 | 说明 |
|------|------------|---------|------|
| Speaker | 80/150 | 8/15 | Speaker音量偏低 |
| A2DP | 120/150 | 12/15 | A2DP音量偏高(绝对音量设备) |
| USB_HEADSET | 60/150 | 6/15 | USB耳机音量中等 |

切换设备时，音量**不会**跟随——每个设备记住自己的历史音量。这避免了"耳机音量突然很大"的问题。

### 13.3.5 Device→Volume联动：音量曲线切换

不同设备类型有独立的音量曲线，设备切换时AudioPolicyManager自动切换曲线：

```mermaid
flowchart TB
    subgraph 音量曲线切换
        DEV_CHANGE[设备切换] --> APM_CURVE[APM选择对应音量曲线]
        APM_CURVE --> HEADSET_CURVE[HEADSET曲线<br>低音量起点更柔和]
        APM_CURVE --> SPEAKER_CURVE[SPEAKER曲线<br>线性或接近线性]
        APM_CURVE --> A2DP_CURVE[A2DP曲线<br>绝对音量设备特殊处理]
    end
    
    subgraph 曲线特征对比
        HEADSET_CURVE --> HC[Headset: -60dB起步<br>前5步缓慢增长<br>保护听力]
        SPEAKER_CURVE --> SC[Speaker: -40dB起步<br>增长更快<br>远距离聆听]
        A2DP_CURVE --> AC[A2DP: 预缩放前3步<br>25%/50%/75%<br>匹配远端增益]
    end
```

**曲线定义**:
- `audio_policy_volumes.xml`: 每个设备类别定义`<volume stream="..." deviceCategory="...">`
- deviceCategory: `HEADSET`, `SPEAKER`, `HEARING_AID`, `A2DP`
- 曲线是`(index, dBAttenuation)`键值对列表

### 13.3.6 Volume→Device联动：安全音量驱动的设备限制

SoundDoseHelper的安全音量检查在音量调节和设备切换时都可能触发设备限制：

```mermaid
flowchart TB
    VOL_RAISE[音量升高请求] --> SDH_CHECK{SoundDoseHelper检查}
    SDH_CHECK -->|超安全阈值| SAFE_WARN[安全音量警告]
    SDH_CHECK -->|未超阈值| NORMAL_SET[正常音量设置]
    
    DEV_SWITCH[设备切换到耳机] --> SDH_INIT{安全音量初始化}
    SDH_INIT -->|设备在安全列表| LIMIT_VOL[限制初始音量到安全值]
    SDH_INIT -->|设备不在安全列表| NO_LIMIT[无限制]
    
    CSD_UPDATE[CSD声剂量更新] --> ATT_CALC{计算CSD衰减dB}
    ATT_CALC --> ATT_POSITIVE[衰减dB>0]
    ATT_CALC --> ATT_ZERO[衰减dB=0]
    
    ATT_POSITIVE --> APPLY_ATT[应用衰减到VolumeStreamState]
    APPLY_ATT --> NEW_MAX[新的有效音量上限]
    
    Note over SAFE_WARN: mPendingVolumeCommand暂存<br>用户确认后执行
    Note over LIMIT_VOL: 安全设备首次连接<br>默认音量=安全索引
```

**安全设备列表**（`mSafeMediaVolumeDevices`）:
- WIRED_HEADSET(0x3)
- WIRED_HEADPHONE(0x4)  
- USB_HEADSET(0x400)
- BLE_HEADSET(0x800)
- BLE_BROADCAST(0x880)
- A2DP_HEADPHONES(0x400)

**安全音量触发时机**:
1. 用户按键升高音量 → `adjustStreamVolume` → `raiseVolumeDisplaySafeMediaVolume()`
2. App调用`setStreamVolume` → `setStreamVolumeWithAttributionInt` → `willDisplayWarningAfterCheckVolume()`
3. CSD声剂量更新 → `postLowerVolumeToRs1()` → 强制降低音量到RS1(80dBA)

### 13.3.7 Focus→Volume→Device联合：导航播报场景

这是车载场景中最典型的三方联动：

```mermaid
sequenceDiagram
    participant USER as 用户
    participant NAV as 导航App
    participant MUSIC as 音乐App
    participant MFC as MediaFocusControl
    participant AS as AudioService
    participant APM as AudioPolicyManager
    participant AF as AudioFlinger
    
    Note over USER: 正在通过A2DP听音乐
    MUSIC->>MFC: 持有STREAM_MUSIC Focus(GAIN)
    MFC-->>MUSIC: Focus已持有
    APM->>APM: MUSIC路由→A2DP
    
    Note over USER: 导航播报开始
    NAV->>MFC: requestFocus(TRANSIENT_MAY_DUCK)
    MFC->>MUSIC: dispatchFocusChange(LOSS_TRANSIENT_CAN_DUCK)
    MUSIC->>MUSIC: 降低音量(duck)
    
    AS->>AF: setVolume(STREAM_MUSIC, duckFactor=0.2)
    Note over AF: 音乐音量=A2DP正常音量×0.2
    
    NAV->>MFC: requestFocus(TRANSIENT_MAY_DUCK, STREAM_MUSIC)
    Note over APM: 导航播报路由→A2DP或Speaker
    
    Note over USER: 导航播报结束
    NAV->>MFC: abandonFocus
    MFC->>MUSIC: dispatchFocusChange(GAIN)
    MUSIC->>MUSIC: 恢复正常音量
    
    AS->>AF: setVolume(STREAM_MUSIC, duckFactor=1.0)
    Note over AF: 音乐音量恢复正常
```

**AAOS Zone隔离方案**:

在AAOS车载系统中，CarAudioService使用Zone机制隔离交互：

| Zone | 用途 | Focus策略 | Volume策略 | Device |
|------|------|----------|-----------|--------|
| Primary Zone | 主娱乐区 | 独立Focus栈 | 独立Volume Group | Speaker/A2DP |
| Navigation Zone | 导航区 | 不与主Zone竞争 | 独立Volume Group | 导航Speaker |
| Call Zone | 通话区 | 不与主Zone竞争 | 独立Volume Group | SCO |

Zone隔离意味着: 导航播报**不duck**音乐，两者独立播放到不同Speaker。

### 13.3.8 Focus→Volume→Device联合：电话来电场景

```mermaid
sequenceDiagram
    participant CALL as 电话App
    participant MUSIC as 音乐App
    participant MFC as MediaFocusControl
    participant AS as AudioService
    participant BROKER as AudioDeviceBroker
    participant APM as AudioPolicyManager
    participant VSS as VolumeStreamState
    
    Note over MUSIC: 正在通过蓝牙耳机听音乐
    MUSIC->>MFC: 持有Focus(GAIN)
    APM->>APM: MUSIC→A2DP
    
    Note over CALL: 来电铃声
    CALL->>MFC: requestFocus(RING, GAIN)
    MFC->>MUSIC: dispatchFocusChange(LOSS)
    MUSIC->>MUSIC: 完全停止播放
    
    AS->>AS: updateStreamVolumeAlias→通话模式
    Note over AS: ALARM→ALARM独立<br>NOTIFICATION→NOTIFICATION独立
    
    APM->>APM: RING→Speaker(外放铃声)
    AS->>VSS: getIndex(SPEAKER, STREAM_RING)
    VSS->>VSS: applyDeviceVolume_syncVSS(SPEAKER)
    
    Note over CALL: 用户接听→SCO
    CALL->>MFC: requestFocus(VOICE_CALL, GAIN)
    BROKER->>APM: SCO设备连接
    APM->>APM: VOICE_CALL→SCO
    APM->>APM: MUSIC→A2DP(优先)或Speaker
    
    Note over CALL: 通话结束
    CALL->>MFC: abandonFocus
    MFC->>MUSIC: dispatchFocusChange(GAIN)
    AS->>AS: updateStreamVolumeAlias→正常模式
    APM->>APM: MUSIC恢复路由→A2DP
    VSS->>VSS: 应用A2DP的历史音量
```

**关键联动点**:
1. **Focus切换**: RING→VOICE_CALL→MUSIC恢复
2. **Device路由**: Speaker(RING)→SCO(VOICE_CALL)→A2DP(MUSIC恢复)
3. **Volume别名**: 正常模式→通话模式→正常模式
4. **per-device音量**: 每个设备的音量独立记忆

### 13.3.9 Device→Focus→Volume联合：蓝牙耳机断开场景

```mermaid
sequenceDiagram
    participant BT as 蓝牙耳机
    participant BROKER as AudioDeviceBroker
    participant APM as AudioPolicyManager
    participant AS as AudioService
    participant MFC as MediaFocusControl
    participant APP as 音乐App
    participant VSS as VolumeStreamState
    
    Note over BT: A2DP耳机正在播放音乐
    APM->>APM: MUSIC→A2DP(活跃路由)
    
    Note over BT: A2DP耳机突然断开
    BT->>BROKER: A2DP断开回调
    BROKER->>AS: checkSendBecomingNoisyIntent
    AS->>APP: ACTION_AUDIO_BECOMING_NOISY广播
    APP->>APP: 暂停播放
    
    Note over AS: 1. BecomingNoisy触发Mute
    AS->>AS: muteMusicStream(mMusicMutedByDevice=true)
    
    Note over BROKER: 2. 设备断开→路由切换
    BROKER->>APM: setDeviceConnectionState(A2DP, UNAVAILABLE)
    APM->>APM: MUSIC→Speaker(路由切换)
    
    Note over AS: 3. 音量切换
    AS->>VSS: applyDeviceVolume_syncVSS(SPEAKER)
    Note over VSS: 使用Speaker历史音量(可能比A2DP低很多)
    
    Note over APP: 4. App暂停→Focus释放
    APP->>MFC: abandonFocus
    MFC->>MFC: Focus栈更新
    
    Note over BT: 蓝牙耳机重新连接
    BT->>BROKER: A2DP连接回调
    BROKER->>APM: setDeviceConnectionState(A2DP, AVAILABLE)
    APM->>APM: MUSIC→A2DP(路由恢复)
    BROKER->>AS: postAccessoryPlugMediaUnmute(A2DP)
    AS->>AS: unmuteMusicStream(mMusicMutedByDevice=false)
    AS->>VSS: applyDeviceVolume_syncVSS(A2DP)
    Note over VSS: A2DP历史音量恢复
```

**BecomingNoisy后Music Mute自动解除**:
- `postAccessoryPlugMediaUnmute()` 在设备重新连接时触发
- 如果`mMusicMutedByDevice=true` → 自动解除Mute
- App需要自己决定是否恢复播放(大多数音乐App收到BecomingNoisy后暂停，不会自动恢复)

### 13.3.10 CSD→Volume→Device联合：声剂量限制场景

```mermaid
sequenceDiagram
    participant HAL as ISoundDose HAL
    participant MP as MelProcessor
    participant MA as MelAggregator
    participant SDH as SoundDoseHelper
    participant AS as AudioService
    participant VSS as VolumeStreamState
    
    Note over HAL: CSD计算流程(Native层)
    HAL->>MP: PCM音频数据
    MP->>MP: A-weighting滤波+MEL计算
    MP->>MA: newMelValues回调
    MA->>MA: 多流MEL合并+CSDDose累积
    MA->>SDH: onNewCsdValue(currentCsd)
    
    Note over SDH: CSD分级警告(Java层)
    SDH->>SDH: currentCsd >= 1.0 → 1X警告Dialog
    SDH->>SDH: currentCsd >= 5.0 → 5X强制降音量
    
    Note over SDH: 5X强制降音量
    SDH->>AS: postLowerVolumeToRs1()
    AS->>VSS: setIndex(RS1安全索引, A2DP_DEVICE)
    VSS->>VSS: applyDeviceVolume_syncVSS(A2DP_DEVICE)
    Note over VSS: 音量被强制限制到RS1=80dBA
    
    Note over SDH: 瞬时暴露检测
    MP->>SDH: onMomentaryExposure(mel>dBA)
    SDH->>SDH: mel > RS2(100dBA) → 瞬时警告
    SDH->>AS: postDisplayCsdWarning(MOMENTARY_EXPOSURE)
    AS->>AS: 弹出瞬时暴露警告Dialog
    
    Note over HAL: CSD衰减计算
    SDH->>SDH: updateCsdAttenuationdB()
    SDH->>SDH: attenuation = f(csdValue, RS1阈值)
    SDH->>AS: MSG_CSD_UPDATE_ATTENUATION
    AS->>VSS: applyCsdAttenuation(attenuationdB)
    VSS->>VSS: 有效音量上限 = maxIndex - attenuationSteps
```

**CSD音量衰减机制**:

SoundDoseHelper计算CSD衰减dB值，并将其应用到VolumeStreamState的有效音量上限：

```
effectiveMaxIndex = mIndexMax - csdAttenuationSteps
```

| CSD值 | 衰减 | 有效音量上限 |
|--------|------|------------|
| 0-0.5 | 0dB | 无限制(mIndexMax) |
| 0.5-1.0 | 渐进衰减 | 逐步降低上限 |
| 1.0(100%) | ~0dB | RS1安全索引(~80dBA) |
| 5.0(500%) | 最大衰减 | 强制降到RS1 |

### 13.3.11 三子系统联动场景汇总

| 场景 | Focus变化 | Device变化 | Volume变化 |
|------|----------|-----------|-----------|
| 导航播报+音乐 | MUSIC→DUCK | 无变化或导航Speaker | duckFactor=0.2 |
| 电话来电 | MUSIC→LOSS | A2DP→Speaker(RING) | RING独立音量 |
| 电话接听 | RING→VOICE_CALL_GAIN | Speaker→SCO | SCO音量+别名切换 |
| 蓝牙耳机断开 | MUSIC可能abandon | A2DP→Speaker | Speaker历史音量+BecomingNoisy Mute |
| 蓝牙耳机连接 | 无变化 | Speaker→A2DP | A2DP历史音量+unmute |
| CSD 1X警告 | 无变化 | 无变化 | 警告Dialog+暂存命令 |
| CSD 5X强制 | 无变化 | 无变化 | 强制降低到RS1 |
| AAOS Zone切换 | Zone内独立Focus | Zone内独立路由 | Zone Volume Group独立 |

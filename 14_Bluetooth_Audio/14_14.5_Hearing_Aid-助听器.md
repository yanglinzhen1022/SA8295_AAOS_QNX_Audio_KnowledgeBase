### 14.5.2 HiSyncId设备组管理

ASHA协议的核心概念是HiSyncId，用于将左右耳助听器配对为一个逻辑组：

```mermaid
flowchart TB
    subgraph HiSyncId设备组
        LEFT[左耳助听器<br>BluetoothDevice A<br>HiSyncId=0x1234]
        RIGHT[右耳助听器<br>BluetoothDevice B<br>HiSyncId=0x1234]
    end

    subgraph 数据结构
        DHM[mDeviceHiSyncIdMap<br>Device→HiSyncId映射<br>A→0x1234, B→0x1234]
        DCM[mDeviceCapabilitiesMap<br>Device→Capabilities映射]
        HCM[mHiSyncIdConnectedMap<br>HiSyncId→IsConnected<br>0x1234→true]
        ADHS[mActiveDeviceHiSyncId<br>当前活动组HiSyncId]
    end

    LEFT --> DHM
    RIGHT --> DHM
    LEFT --> DCM
    RIGHT --> DCM
    DHM --> HCM
```

**HiSyncId关键逻辑**：

| 场景 | 行为 |
|------|------|
| 左右耳配对 | 两者具有相同HiSyncId，可同时控制 |
| 单耳使用 | 只有一个设备活跃，HiSyncId仍有效 |
| 切换活动设备 | mActiveDeviceHiSyncId更新，同组设备同步切换 |
| 断开同组设备 | mHiSyncIdConnectedMap标记为false |

**HiSyncId特殊值**：`BluetoothHearingAid.HI_SYNC_ID_INVALID`表示设备不属于任何组。

### 14.5.3 HearingAidStateMachine状态机

```mermaid
stateDiagram-v2
    [*] --> Disconnected
    Disconnected --> Connecting : CONNECT消息
    Connecting --> Connected : STACK_EVENT:CONNECTED
    Connecting --> Disconnected : CONNECT_TIMEOUT
    Connected --> Disconnecting : DISCONNECT消息
    Disconnecting --> Disconnected : STACK_EVENT:DISCONNECTED
```

与A2DP/LE Audio采用相同的四状态模型，但助听器场景更注重：
- 自动重连（配对后开机自动连接）
- 双耳同步（同一HiSyncId的设备同时连接/断开）

### 14.5.4 ASHA音量机制 — 增益dB

助听器使用dB增益值而非百分比音量，范围-128dB到0dB：

```mermaid
flowchart TB
    VOL[音量调节KeyEvent或API] --> HA_CHECK{当前路由是Hearing Aid?}
    HA_CHECK -->|是| HA_VOL[postSetHearingAidVolumeIndex]
    HA_CHECK -->|否| NORMAL[正常音量路径]

    HA_VOL --> BTH_SET[BtHelper.setHearingAidVolume]
    BTH_SET --> DB_CALC[AudioSystem.getStreamVolumeDB<br>计算dB增益值]
    DB_CALC --> RANGE_CHECK{gainDB < -128dB?}
    RANGE_CHECK -->|是| CLAMP[gainDB = -128<br>BT_HEARING_AID_GAIN_MIN]
    RANGE_CHECK -->|否| OK[使用计算值]
    CLAMP --> SET[mHearingAid.setVolume<br>gainDB: -128~0dB]
    OK --> SET
    SET --> ASHA[ASHA协议栈<br>助听器端执行增益]
```

**BtHelper.setHearingAidVolume()实现**（源码[`BtHelper.java:392-420`](frameworks/base/services/core/java/com/android/server/audio/BtHelper.java:392)）：

```java
synchronized void setHearingAidVolume(int index, int streamType,
        boolean isHeadAidConnected) {
    if (mHearingAid == null) return;
    // 助听器期望-128dB到0dB范围的增益值
    int gainDB = (int) AudioSystem.getStreamVolumeDB(streamType, index / 10,
            AudioSystem.DEVICE_OUT_HEARING_AID);
    if (gainDB < BT_HEARING_AID_GAIN_MIN) {
        gainDB = BT_HEARING_AID_GAIN_MIN;  // -128
    }
    mHearingAid.setVolume(gainDB);
}
```

**音量范围对比**：

| Profile | 音量范围 | 单位 | 接口方法 |
|---------|----------|------|----------|
| A2DP (AVRCP) | 0-127 | 绝对音量 | mA2dp.setAvrcpAbsoluteVolume() |
| LE Audio (VCP) | 0-255 | 绝对音量 | mLeAudio.setVolume() |
| Hearing Aid (ASHA) | -128~0 | dB增益 | mHearingAid.setVolume() |
| HFP (SCO) | 0.0-1.0 | 归一化浮点 | IBluetooth.setHfpConfig() |

### 14.5.7 ASHA与LE Audio的关系

ASHA和LE Audio都使用BLE传输音频，但属于不同协议：

```mermaid
flowchart TB
    subgraph BLE音频协议
        ASHA[ASHA协议<br>Apple主导<br>GATT-based<br>低延迟低功耗]
        LEA[LE Audio/BAP<br>Bluetooth SIG标准<br>ISO-based<br>多功能高音质]
    end

    subgraph 共同特征
        BLE[使用BLE无线电]
        LOW_LATENCY[低延迟传输]
        GATT_SERVER[GATT服务器]
    end

    ASHA --> BLE
    LEA --> BLE
    ASHA --> LOW_LATENCY
    ASHA --> GATT_SERVER
```

| 特性 | ASHA | LE Audio |
|------|------|----------|
| 主导方 | Apple | Bluetooth SIG |
| 传输通道 | GATT | ISO (CIS/BIS) |
| 编解码 | G.722 (Opus可选) | LC3 |
| 延迟 | 极低(适合助听器) | 低(适合多媒体) |
| 双向音频 | 不支持 | 支持(CIS双向) |
| 广播 | 不支持 | 支持(BIS) |
| 设备类型 | DEVICE_OUT_HEARING_AID | DEVICE_OUT_BLE_HEADSET |

### 14.5.9 AAOS车载Hearing Aid场景

| 场景 | 实现方式 | 关键点 |
|------|----------|--------|
| 驾驶员助听器导航 | DEVICE_OUT_HEARING_AID路由 | 车载导航音频传到助听器 |
| 助听器+车载扬声器并行 | 双输出路由 | 助听器和Speaker同时播放 |
| 助听器音量独立调节 | dB增益模式 | -128~0dB范围，getStreamVolumeDB映射 |
| 听障辅助系统 | ASHA协议+车载麦克风 | 车载麦克风拾音→助听器播放 |

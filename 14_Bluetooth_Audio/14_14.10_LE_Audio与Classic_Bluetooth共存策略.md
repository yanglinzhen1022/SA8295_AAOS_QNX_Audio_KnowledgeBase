### 14.10.1 双模共存架构概述

Android 14同时支持Classic Bluetooth(A2DP/HFP)和LE Audio，需要协调两者在设备选择、音频路由、Suspend互斥等方面的冲突。AudioDeviceBroker和BtHelper是共存策略的核心实现。

源码路径：
- [`AudioDeviceBroker.java`](frameworks/base/services/core/java/com/android/server/audio/AudioDeviceBroker.java) — Suspend互斥与设备路由
- [`BtHelper.java`](frameworks/base/services/core/java/com/android/server/audio/BtHelper.java) — Profile代理管理
- [`AudioPolicyManager.cpp`](frameworks/av/services/audiopolicy/managerdefault/AudioPolicyManager.cpp) — 设备选择策略

```mermaid

        BTH[BtHelper<br>Profile代理管理]
    end

    subgraph Classic Bluetooth

        HFP[HFP Service<br>通话语音]
    end

    subgraph LE Audio

    BTH --> A2DP
    BTH --> HFP
    BTH --> LEA
    BTH --> HA
```

### 14.10.2 Suspend互斥机制

AudioDeviceBroker通过`mBluetoothAudioStateLock`保护的8个状态变量实现共存互斥（[`AudioDeviceBroker.java:893-921`](frameworks/base/services/core/java/com/android/server/audio/AudioDeviceBroker.java:893)）：

```mermaid
stateDiagram-v2
    [*] --> Idle: 初始状态
    Idle --> A2DP_Active: A2DP连接+播放
    Idle --> LE_Active: LE Audio连接+播放
    Idle --> SCO_Active: SCO通话开始

    A2DP_Active --> A2DP_Suspended_By_SCO: SCO激活<br>强制A2DP Suspend
    LE_Active --> LE_Suspended_By_SCO: SCO激活<br>强制LE Suspend

    A2DP_Active --> A2DP_Suspended_By_LE: LE Audio切换<br>主动Suspend A2DP
    LE_Active --> LE_Suspended_By_A2DP: A2DP切换<br>主动Suspend LE Audio

    SCO_Active --> SCO_Active_A2DP_Suspended: SCO进行中<br>A2DP保持Suspend
    SCO_Active --> SCO_Active_LE_Suspended: SCO进行中<br>LE保持Suspend

    A2DP_Suspended_By_SCO --> A2DP_Active: SCO结束<br>恢复A2DP
    LE_Suspended_By_SCO --> LE_Active: SCO结束<br>恢复LE Audio

    SCO_Active_A2DP_Suspended --> A2DP_Active: SCO结束
    SCO_Active_LE_Suspended --> LE_Active: SCO结束
```

**8个状态变量详解**（[`AudioDeviceBroker.java:894-921`](frameworks/base/services/core/java/com/android/server/audio/AudioDeviceBroker.java:894)）：

| 变量 | 类型 | 说明 |
|------|------|------|
| mBluetoothScoOn | boolean | SCO是否激活(外部+内部) |
| mBluetoothScoOnApplied | boolean | SCO状态已同步到HAL |
| mBluetoothA2dpSuspendedExt | boolean | 外部A2DP Suspend请求 |
| mBluetoothA2dpSuspendedInt | boolean | 内部A2DP Suspend请求 |
| mBluetoothA2dpSuspendedApplied | boolean | 合并后A2DP Suspend已同步到HAL |
| mBluetoothLeSuspendedExt | boolean | 外部LE Audio Suspend请求 |
| mBluetoothLeSuspendedInt | boolean | 内部LE Audio Suspend请求 |
| mBluetoothLeSuspendedApplied | boolean | 合并后LE Audio Suspend已同步到HAL |

### 14.10.3 updateAudioHalBluetoothState互斥核心

[`updateAudioHalBluetoothState()`](frameworks/base/services/core/java/com/android/server/audio/AudioDeviceBroker.java:934) 是共存互斥的核心方法：

```
方法逻辑(行934-991):
1. synchronized(mBluetoothAudioStateLock)
2. 计算scoOnApplied = mBluetoothScoOn (SCO是否激活)
3. 计算a2dpSuspendedApplied = mBluetoothA2dpSuspendedExt || mBluetoothA2dpSuspendedInt
4. 计算leSuspendedApplied = mBluetoothLeSuspendedExt || mBluetoothLeSuspendedInt
5. if(scoOnApplied) → 强制a2dpSuspendedApplied = true, leSuspendedApplied = true
   // SCO优先级最高：通话时强制Suspend所有音乐
6. 检查状态是否变化(对比Applied值)
7. if(变化) → AudioSystem.setParameters()同步三参数:
   - BT_SCO=on/off
   - A2dpSuspended=true/false
   - LeAudioSuspended=true/false
8. 更新mBluetoothXxxApplied值
```

**SCO优先互斥规则**：

```mermaid
flowchart TB
    SCO_CHECK{SCO是否激活?}
    SCO_YES[SCO=True<br>强制A2DP Suspend=True<br>强制LE Suspend=True<br>BT_SCO=on]
    SCO_NO[SCO=False<br>独立检查A2DP Suspend<br>独立检查LE Suspend<br>BT_SCO=off]

    SCO_CHECK -->|是| SCO_YES
    SCO_CHECK -->|否| SCO_NO

    SCO_NO --> A2DP_CHECK{A2DP Suspend?<br>Ext OR Int}
    A2DP_CHECK -->|true| A2DP_SUS[A2dpSuspended=true]
    A2DP_CHECK -->|false| A2DP_ACTIVE[A2dpSuspended=false]

    SCO_NO --> LE_CHECK{LE Suspend?<br>Ext OR Int}
    LE_CHECK -->|true| LE_SUS[LeAudioSuspended=true]
    LE_CHECK -->|false| LE_ACTIVE[LeAudioSuspended=false]
```

### 14.10.4 设备切换策略

当用户切换蓝牙设备时，`queueOnBluetoothActiveDeviceChanged()`(行860-890)处理路由切换：

```mermaid
sequenceDiagram
    participant BT as BT Service
    participant BTH as BtHelper
    participant ADB as AudioDeviceBroker
    participant APS as AudioPolicyService
    participant AF as AudioFlinger

    BT->>BTH: ACTION_ACTIVE_DEVICE_CHANGED
    BTH->>ADB: onBluetoothActiveDeviceChanged(profile, device)
    ADB->>ADB: queueOnBluetoothActiveDeviceChanged()

    alt 设备变化(新设备)
        ADB->>ADB: createBtDeviceInfo(device, profile)
        ADB->>ADB: setA2dpSuspended(true) 暂停旧A2DP
        ADB->>ADB: setLeAudioSuspended(true) 暂停旧LE
        ADB->>ADB: handleBtDeviceChanged(newBtInfo)
        ADB->>APS: setDeviceConnectionState()
        APS->>AF: 重新打开输出流
        ADB->>ADB: setA2dpSuspended(false) 恢复新A2DP
        ADB->>ADB: setLeAudioSuspended(false) 恢复新LE
    else 同设备更新
        ADB->>ADB: handleBtDeviceConfigChanged()
        Note over ADB: 仅更新Codec/配置
    end
```

**Profile→AudioDevice映射**（[`createBtDeviceInfo()`](frameworks/base/services/core/java/com/android/server/audio/AudioDeviceBroker.java:812)）：

| Profile | AudioDevice类型 | 说明 |
|---------|----------------|------|
| A2DP | DEVICE_OUT_BLUETOOTH_A2DP | Classic音乐 |
| HFP | DEVICE_OUT_BLUETOOTH_SCO | 通话语音 |
| LE Audio | DEVICE_OUT_BLE_HEADSET | LE音乐+通话 |
| Hearing Aid | DEVICE_OUT_HEARING_AID | ASHA助听器 |
| LE Broadcast | DEVICE_OUT_BLE_BROADCAST | LE广播 |

### 14.10.5 Active Device管理

Android 14蓝牙音频设备通过"Active Device"机制管理：

```mermaid
flowchart TB
    subgraph BtHelper Active Device管理
        A2DP_ACT[A2DP Active Device<br>mA2dp.getActiveDevices]
        LE_ACT[LE Audio Active Device<br>mLeAudio.getActiveDevices]
        HFP_ACT[HFP Active Device<br>mBluetoothHeadset.getActiveDevice]
        HA_ACT[HearingAid Active Device<br>mHearingAid.getActiveDevices]
    end

    subgraph 优先级规则
        P1[1. SCO通话 > 一切<br>通话中强制Suspend其他]
        P2[2. 最后连接的设备<br>成为Active Device]
        P3[3. 手动切换<br>用户在设置中切换]
        P4[4. AAOS车载<br>车机优先于手机]
    end

    A2DP_ACT --> P2
    LE_ACT --> P2
    HFP_ACT --> P1
    HA_ACT --> P2
```

**A2DP与LE Audio切换冲突**：

| 场景 | 行为 | Suspend状态 |
|------|------|-------------|
| A2DP连接 → LE Audio连接 | LE Audio成为Active | A2DP Suspend |
| LE Audio连接 → A2DP连接 | A2DP成为Active | LE Suspend |
| A2DP+LE同时连接 | 最后连接者优先 | 另一个Suspend |
| SCO来电 | SCO接管 | A2DP+LE双Suspend |
| SCO结束 | 恢复Suspend前状态 | A2DP/LE恢复 |

### 14.10.6 Suspend API双标志机制

[`setA2dpSuspended()`](frameworks/base/services/core/java/com/android/server/audio/AudioDeviceBroker.java:1004) 和 [`setLeAudioSuspended()`](frameworks/base/services/core/java/com/android/server/audio/AudioDeviceBroker.java:1034) 使用Ext/Int双标志：

```
setA2dpSuspended(suspend, fromExternal):
  synchronized(mBluetoothAudioStateLock):
    if(fromExternal):
      mBluetoothA2dpSuspendedExt = suspend
    else:
      mBluetoothA2dpSuspendedInt = suspend
    // Ext OR Int → 合并结果
    updateAudioHalBluetoothState()

setLeAudioSuspended(suspend, fromExternal):
  synchronized(mBluetoothAudioStateLock):
    if(fromExternal):
      mBluetoothLeSuspendedExt = suspend
    else:
      mBluetoothLeSuspendedInt = suspend
    updateAudioHalBluetoothState()
```

**双标志来源**：

| 标志 | 来源 | 说明 |
|------|------|------|
| Ext(外部) | AudioService公共API | 应用/系统设置调用 |
| Int(内部) | AudioDeviceBroker内部 | SCO互斥/设备切换时自动触发 |
| OR合并 | Ext OR Int | 任一为true即Suspend |

### 14.10.8 通话场景共存策略

通话是双模共存最复杂的场景：

```mermaid
sequenceDiagram
    participant USER as 用户
    participant HFP as HFP Service
    participant LEA as LeAudioService
    participant ADB as AudioDeviceBroker
    participant HAL as Audio HAL

    USER->>HFP: 来电
    HFP->>ADB: onScoAudioStateChanged(SCO_STATE_ACTIVATE_REQ)
    ADB->>ADB: mBluetoothScoOn = true
    ADB->>ADB: updateAudioHalBluetoothState()

    Note over ADB,HAL: SCO优先互斥触发
    ADB->>ADB: 强制a2dpSuspendedApplied=true
    ADB->>ADB: 强制leSuspendedApplied=true
    ADB->>HAL: BT_SCO=on<br>A2dpSuspended=true<br>LeAudioSuspended=true

    Note over ADB,HAL: 通话进行中...
    Note over HFP,LEA: A2DP和LE Audio保持Suspend

    USER->>HFP: 挂断
    HFP->>ADB: onScoAudioStateChanged(SCO_STATE_DEACTIVATE_REQ)
    ADB->>ADB: mBluetoothScoOn = false
    ADB->>ADB: updateAudioHalBluetoothState()

    Note over ADB,HAL: 恢复Suspend前状态
    ADB->>ADB: 恢复a2dpSuspendedApplied
    ADB->>ADB: 恢复leSuspendedApplied
    ADB->>HAL: BT_SCO=off<br>A2dpSuspended=原值<br>LeAudioSuspended=原值
```

**LE Audio通话 vs HFP通话**：

| 维度 | HFP通话 | LE Audio通话 |
|------|---------|-------------|
| Profile | HFP(Hands-Free) | BAP+CAP(Call Mode) |
| 编解码器 | CVSD/mSBC | LC3 |
| 音频设备 | DEVICE_OUT_BLUETOOTH_SCO | DEVICE_OUT_BLE_HEADSET |
| SCO状态机 | 6状态(行92-103) | 不使用SCO |
| 双向音频 | SCO双向CIS | CIS双向 |
| 音质 | mSBC: 16kHz/32kbps | LC3: 32kHz/64kbps |
| Suspend影响 | Suspend A2DP+LE | 不影响A2DP |

### 14.10.9 AudioPolicy设备选择策略

AudioPolicyManager在设备选择时考虑双模优先级：

```mermaid
flowchart TB
    START[音频流请求设备]
    CHECK_MODE{AudioMode?}

    CHECK_MODE -->|MODE_IN_CALL| SCO[SCO设备优先<br>DEVICE_OUT_BLUETOOTH_SCO<br>或DEVICE_OUT_BLE_HEADSET]
    CHECK_MODE -->|MODE_NORMAL| MEDIA[媒体设备选择]

    MEDIA --> CHECK_LE{LE Audio可用?}
    CHECK_LE -->|是| CHECK_A2DP{A2DP可用?}
    CHECK_A2DP -->|是| LAST[最后连接设备优先]
    CHECK_A2DP -->|否| LE_ONLY[LE Audio设备]
    CHECK_LE -->|否| CHECK_A2DP2{A2DP可用?}
    CHECK_A2DP2 -->|是| A2DP_ONLY[A2DP设备]
    CHECK_A2DP2 -->|否| SPEAKER[扬声器回退]

    LAST --> CHECK_SUSPEND{Suspend状态?}
    CHECK_SUSPEND -->|A2DP Suspend| LE_ROUTE[路由到LE Audio]
    CHECK_SUSPEND -->|LE Suspend| A2DP_ROUTE[路由到A2DP]
```

### 14.10.10 AAOS车载双模共存场景

| 场景 | Classic BT | LE Audio | 共存策略 |
|------|-----------|----------|----------|
| 驾驶员手机通话 | HFP SCO | 不使用 | SCO优先 |
| 驾驶员听导航 | A2DP Suspend | LE Audio | LE优先(低延迟) |
| 乘客后排听音乐 | A2DP | 不使用 | Classic BT |
| 全车Auracast广播 | 不使用 | Broadcast | LE Broadcast |
| 助听器辅助 | 不使用 | LE Audio+ASHA | LE/ASHA独立 |
| 手机-车机双连接 | A2DP+HFP | LE Audio | 双模并存 |
| 通话导航混合 | HFP SCO | 导航LE | SCO+非Suspend混音 |
